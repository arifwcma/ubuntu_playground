CRITICAL THINGS TO KNOW (hard-won, do not repeat mistakes)

1. The qgis/qgis-server:ltr container has a custom nginx.conf mounted at
   /home/ssm-user/apps/qgis-server/nginx/nginx.conf
   This is required because the default rewrite only matched /ows/ exactly.
   Do NOT remove this volume mount.

2. QGIS server requires these env vars (already in compose.yaml):
   - QGIS_SERVER_LIZMAP_REVEAL_SETTINGS=true
   - QGIS_PLUGINPATH=/opt/qgis-server/plugins

3. Lizmap server plugin must be >= 2.13.0. Currently 2.14.1 at:
   /home/ssm-user/apps/qgis-server/plugins/lizmap_server/
   Plugin is owned by ssm-user. If you ever re-extract it, use sudo then chown back to ssm-user.

4. Lizmap 3.9 defaults to requiring Py-QGIS-Server. We use plain FCGI.
   allowFcgi=on is set under [qgisWrapper] in:
   /home/ssm-user/apps/lizmap/var/config/localconfig.ini.php
   Do NOT remove this.

5. Nginx rate limit for wfml.wcma.work is 60r/s with burst=100.
   Lizmap fires 20+ parallel requests on page load. Lower values cause 503s.

6. Lizmap bind mounts (not named volumes) are at /home/ssm-user/apps/lizmap/var/ and /home/ssm-user/apps/lizmap/www/. Config, DB, and logs live there. The var/ directory has a .gitignore to exclude runtime files.
